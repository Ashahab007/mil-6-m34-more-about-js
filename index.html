Javascript Execution Context & Callstack

Two kinds of executional Context in JS

  1. Global Execution Context (GEC)

    2. যখন javascript এর কোন file run করা হয় তখন default ভাবে Global Execution Context তৈরি করে। সেখানে code গুলা run হয়। যদি কোন function এর body থাকে তাহলে সেটা Global Context এ assign হয় কিন্তু call হয় না।

  2. Function Execution Context (FEC)

    3. যখন function call করা হয় তখন function execution Context তৈরি হয়।


How all the execution context executes?

Creation Phase

যখন উপরের GEC এবং FEC তৈরি হয় তা javascript এর creation phase তৈরি করে যা কোন code execution করে না। শুধু variable এবং function এর body গুলা execution context এ assign করে।

Execution Phase

Assigns the values to variables and executes the function call


Javascript is a single threaded language

Javascript performs only one task at a time. It has only one call stack.

Synchronous & Asynchronous

Synchronous: Executes immediately

Asynchronous: Executes after specific time intervals. e.g. fetch, setTimeout.
